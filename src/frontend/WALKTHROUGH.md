# AgenticFleet Frontend Walkthrough

This guide provides an overview of the AgenticFleet frontend application, its structure, and how to use it.

## Application Structure

The application is built with React, TypeScript, and Vite, following a modern component-based architecture.

### Key Directories

- **`src/components/chat/`**: Core chat interface components.
  - `ChatContainer.tsx`: The main chat view manager.
  - `ConversationSidebar.tsx`: Manages the list of conversations.
  - `MessageList.tsx`: Renders the scrollable list of messages.
  - `MessageInput.tsx`: The user input area.
- **`src/stores/`**: State management using Zustand.
  - `chatStore.ts`: Central store for conversations and messages.
- **`src/lib/api/`**: API integration.
  - `chatApi.ts`: Client for backend chat endpoints.
- **`src/hooks/`**: Custom React hooks.
  - `useStreamingChat.ts`: Handles SSE (Server-Sent Events) streaming for real-time responses.

## User Interface Overview

### 1. Sidebar (Left Panel)

- **New Conversation**: Click the "New Conversation" button to start a fresh chat context.
- **Conversation History**: Lists previous conversations. Click on any item to switch to that chat.
- **Metadata**: Shows the number of messages in each conversation.

### 2. Chat Area (Main Panel)

- **Message List**: Displays the conversation history.
  - **User Messages**: Aligned to the right.
  - **Assistant Messages**: Aligned to the left. Supports Markdown rendering.
  - **Streaming**: Responses appear in real-time as they are generated by the agent.
  - **Orchestrator Thoughts**: Intermediate thoughts from the backend orchestrator (if enabled) are displayed to show the agent's reasoning process.
- **Input Area**: Text field to send messages to the agent. Press `Enter` to send, `Shift+Enter` for new lines.

## Developer Guide

### Running the Application

1. **Start the Backend**:
   Ensure the FastAPI backend is running on port 8000.

   ```bash
   make dev  # Runs both backend and frontend
   # OR
   make backend
   ```

2. **Start the Frontend**:

   ```bash
   cd src/frontend
   npm install
   npm run dev
   ```

3. **Running Tests**:
   ```bash
   npm test
   ```

### UI Components

The application uses a mix of custom components and `@openai/apps-sdk-ui` (or similar UI libraries) for a consistent look and feel. Tailwind CSS is used for styling.

### State Management

`chatStore` handles the application state. It fetches conversations on mount and updates the UI optimistically when messages are sent.

```typescript
const { addMessage, currentConversationId } = useChatStore();
addMessage(currentConversationId, "user", "Hello!");
```

### Streaming Integration

The `useStreamingChat` hook abstracts the complexity of Server-Sent Events. It parses the incoming stream and distinguishes between content deltas and orchestrator events.

```typescript
const { sendMessage, isStreaming } = useStreamingChat({ ... })
await sendMessage('My prompt')
```
